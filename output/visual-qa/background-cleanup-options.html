CreatorBridge image cleanup

The issue is not the folder structure. It is image intent.

The current implementation treats photos like decoration. The cleanup should make every image serve one of three jobs: identify the production pillar, create atmospheric continuity, or prove client/creator work.

Current risk

Pictures as filler

Keep placing photos section-by-section

This keeps the folder organization but does not solve mismatches, hard seams, or random image energy.

×Post Production can accidentally get camera/lens imagery instead of editing, audio, color, or motion visuals.
×Full-width sections reveal hard image cutoffs when backgrounds change.
×Looks like separate photos pasted onto a design instead of one cinematic page.
Avoid
Recommended

Slot map + blend rules

Give every image a role and enforce it in code

Use the images intentionally: video = cameras/sets, photo = studio/product/portrait, post = editing console/audio/color/motion.

✓Correct pillar images: Post Production uses editing/audio/color imagery, not generic camera photos.
✓Add dark scrims, gradient masks, and section fade overlaps so the page feels continuous.
✓Keep the hero camera-rig shot, but make lower-page imagery quieter and more atmospheric.
Best path
Safer but flatter

Ambient only

Remove most visible background photos

This solves the mess quickly by making backgrounds almost invisible and relying on cards for imagery.

✓Fastest way to remove hard seams and inappropriate images.
✓Less legal and visual risk while placeholder imagery is temporary.
×Loses some of the premium cinematic depth Claude Design was aiming for.
